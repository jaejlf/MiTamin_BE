💊 My Tamin API Document

참고사항

  • request 속성의 description에 '*'문자가 붙어있지 않다면 필수값이 아니다.
    이 경우 데이터가 없다면 아예 전송하지 않아야 서버에서 설정한 default값으로 저장된다.

  • ex. 회원가입 시 유저가 마이타민 섭취 지정 시간 설정 '건너뛰기’를 했을경우,

{
  "email" : "mytamin@naver.com",
  "password" : "password1234",
  "nickname" : "강철멘탈"
}

⇒ 위와 같이, 데이터가 없는 mytaminHour, mytaminMin 속성은 전송하지 않는다.

  • ⭐ 만약 '*' 문자가 붙어있는데 전송할 값이 존재하지 않는다면, 문서에 기술된 기본값으로 세팅하여 전송한다.

Auth

회원 가입

  • mytaminHour은 24시간 단위의 값으로 요청한다. (ex. 오후 3시 → "15")

HTTP request

POST /auth/signup HTTP/1.1
Content-Type: application/json;charset=UTF-8
Content-Length: 150
Host: MY-TAMIN-URL

{
  "email" : "mytamin@naver.com",
  "password" : "password1234",
  "nickname" : "강철멘탈",
  "mytaminHour" : "22",
  "mytaminMin" : "00"
}

HTTP response

HTTP/1.1 201 Created
Content-Type: application/json;charset=UTF-8
Content-Length: 283

{
  "statusCode" : 201,
  "message" : "회원 가입",
  "data" : {
    "email" : "mytamin@naver.com",
    "nickname" : "강철멘탈",
    "profileImgUrl" : null,
    "beMyMessage" : "마음 면역력이 높아질",
    "mytaminHour" : "22",
    "mytaminMin" : "00"
  }
}

토큰 재발급

  • 새로 발급한 accessToken과 기존의 refreshToken을 리턴한다.

    • refreshToken의 만료 기간이 2일 이내로 남았을 경우 refreshToken 또한 새로 발급한다.

  • 만약 403 응답(= INVALID_TOKEN_ERROR)일 경우에는 재로그인하도록 한다.

GET /auth/reissue HTTP/1.1
Content-Type: application/json;charset=UTF-8
Content-Length: 78
Host: MY-TAMIN-URL

{

  "refreshToken" : "{{REFRESH_TOKEN}}"
}

HTTP/1.1 200 OK
Content-Type: application/json;charset=UTF-8
Content-Length: 164

{
  "statusCode" : 200,
  "message" : "토큰 재발급",
  "data" : {
    "accessToken" : "{{ACCESS_TOKEN}}",
    "refreshToken" : "{{REFRESH_TOKEN}}"

}

Home

웰컴 메세지

  • 조건에 따라 웰컴 메시지를 리턴한다.

  • 조건 1, 2, 3 순으로 높은 우선 순위가 적용된다.

  • 조건

    • 조건 1. 마이타민 섭취 완료

      • 오늘 하루도 수고 많았어요 :)

    • 조건 2. 마이타민 섭취 지정 시간 및 섭취 여부

      • 섭취 지정 시간 2시간 전 : 오늘의 마이타민 섭취를 잊지마세요 :)

      • 섭취 지정 시간 2시간 후 : 마이타민 섭취.. 잊으시면 안 돼요 !

    • 조건 3. 시간에 따른 웰컴 메세지

      • am 5:00 ~ am 11:59 : 오늘도 힘차게 시작해볼까요 ?

      • pm 12:00 ~ pm 18:59 : 어떤 하루를 보내고 계신가요 ?

      • pm 19:00 ~ am 4:59 : 푹 쉬고 내일 만나요

  • 하루 진단 or 칭찬 처방 중 완료하지 않은 행동이 있다면 null로 리턴된다. (ex. data.report : null)

    • report.canEdit, care.canEdit은 수정 가능할 경우 true, 불가능할 경우 false를 리턴한다.

    • 4 : "좋은 편이에요 :)"

  • 년/월을 기준으로 오름차순 정렬되어 리턴된다.

  • performedAt은 이미지가 존재하지 않을 경우 대체 제목으로 사용되는 값이다.

GET /daynote/list HTTP/1.1

데이노트 작성 가능 여부

  • 사용자가 선택한 데이노트 작성 날짜를 yyyy.MM 형식으로 요청한다. (ex. 22년 1월 → 2022.01)

  • performedAt에 해당하는 날짜에 데이노트 작성이 가능하면 true, 불가능하면 false를 리턴한다.

  • (아직 화면설계서에 반영되지 않았지만) 기획팀과 상의된 플로우

    • 1. 데이노트 작성 가능 여부 체크

    • 1-1. 작성 가능하다면(= true) 데이노트 작성 화면으로 넘어가기

    • 1-2. 작성 불가능하다면(= false, 이미 해당 날짜에 데이노트가 존재한다면) 데이노트 수정/삭제 여부를 묻기

데이노트 작성

  • Multipart 타입으로 이미지 리스트를 전송한다.

  • fileList 내 이미지의 확장자는 .png, .jpg, .jpeg 또는 null 중 하나여야 한다.

  • 이미지는 최대 5장까지 첨부할 수 있다. (아직 반영 전)

$ http --multipart POST 'http://MY-TAMIN-URL/daynote/new' \
    'fileList'@'[org.springframework.mock.web.MockMultipartFile@3217aada, org.springframework.mock.web.MockMultipartFile@34f14035]' \
    'X-AUTH-TOKEN:{{ACCESS_TOKEN}}' \
    'wishText=빵 나오는 시간에 맞춰서 갓 나온 빵 사기' \
    'note=따끈따끈한 빵을 샀다. 맛있었따 :0' \
    'performedAt=2022.10'
